# 12단계: 슬래시 명령어 활용하기

> 이 챕터에서 할 것: Claude Code에 내장된 슬래시 명령어를 익혀서 자주 쓰는 기능을 빠르게 실행합니다.

> ⚠️ 먼저 [6단계: Claude Code 시작하기](06.Claude-Code-시작하기.md)를 완료해야 합니다.

---

## 12-1. 슬래시 명령어란?

Claude Code 대화창에서 `/`를 입력하면 내장된 기능들을 바로 실행할 수 있습니다.  
텍스트로 "대화 내용을 요약해줘"라고 요청하는 것보다 `/compact`를 입력하는 게 훨씬 빠릅니다.

슬래시 명령어는 세 종류입니다:

| 종류 | 설명 | 예시 |
|------|------|------|
| **내장 명령어** | Claude Code가 기본으로 제공 | `/help`, `/clear`, `/compact` |
| **커스텀 명령어** | `.claude/commands/`에 파일로 저장 | `/배포`, `/테스트` |
| **Skill 명령어** | `.claude/skills/`에 파일로 저장 (심화) | `/코드-설명`, `/버그-찾기` |

이 챕터에서는 내장 명령어와 커스텀 명령어를 다룹니다.  
Skill 명령어에 대한 자세한 내용은 [10단계](10.Skill-md-작성하기.md)를 참고하세요.

---

## 12-2. 자주 쓰는 명령어

### `/help` — 도움말 보기

사용할 수 있는 모든 명령어와 Skill 목록을 보여줍니다.

```
/help
```

---

### `/clear` — 대화 초기화

현재 대화 내용을 모두 지우고 새로 시작합니다.  
AI가 이전 대화 내용을 "기억"하지 않아야 할 때 사용합니다.

```
/clear
```

---

### `/compact` — 대화 압축

대화가 길어지면 AI가 느려지거나 앞 내용을 잊을 수 있습니다.  
`/compact`를 실행하면 대화 내용을 요약해서 가볍게 만들어 줍니다.

```
/compact
```

> 💡 대화가 많이 쌓였다고 느껴지면 `/compact`를 실행하세요.

---

### `/recap` — 대화 요약 보기

지금까지 어떤 작업을 했는지 요약해서 보여줍니다.  
오래 작업하다가 "내가 뭘 하고 있었지?" 싶을 때 유용합니다.

```
/recap
```

---

### `/config` — 설정 변경

AI 모델, 화면 테마, 권한 등 Claude Code의 설정을 바꿀 수 있습니다.

```
/config
```

---

### `/mcp` — MCP 서버 관리

연결된 MCP 서버(외부 도구 연결) 목록을 보거나 관리합니다.  
MCP에 대한 자세한 내용은 [11단계](11.유용한-플러그인-소개.md)를 참고하세요.

```
/mcp
```

---

### `/plugin` — 플러그인 관리

설치된 플러그인을 확인하거나 새 플러그인을 설치할 수 있습니다.

```
/plugin
```

---

### `/memory` — 메모리 확인

Claude Code가 기억하고 있는 내용(CLAUDE.md 등)을 확인하고 수정합니다.

```
/memory
```

---

### `/review` — 코드 리뷰

현재 변경한 코드를 AI가 검토해서 문제점을 알려줍니다.

```
/review
```

---

### `/init` — CLAUDE.md 초기화

프로젝트를 분석해서 CLAUDE.md 파일을 자동으로 만들어 줍니다.  
CLAUDE.md에 대한 자세한 내용은 [9단계](09.CLAUDE-md-작성하기.md)를 참고하세요.

```
/init
```

---

## 12-3. 나만의 명령어 만들기

자주 쓰는 작업을 커스텀 명령어로 만들면 `/명령어이름`으로 바로 실행할 수 있습니다.

### 저장 위치

`.claude/commands/` 폴더에 마크다운(`.md`) 파일을 만들면 됩니다.

```
my-todo-app/
└── .claude/
    ├── commands/    ← 커스텀 명령어 저장 위치
    └── skills/      ← Skill 저장 위치
```

### 파일 만들기

`.claude/commands/` 폴더 안에 `테스트.md` 파일을 만들고 아래처럼 입력하세요:

```markdown
현재 프로젝트의 모든 테스트를 실행하고, 실패한 항목이 있으면 원인과 해결 방법을 알려주세요.
```

### 사용하기

Claude Code 대화창에서 파일 이름으로 바로 호출합니다:

```
/테스트
```

### 커스텀 명령어 예시

**배포 전 점검** (`.claude/commands/배포-점검.md`)
```markdown
배포 전에 아래 항목을 순서대로 점검해 주세요:
1. 콘솔 오류 메시지가 있는지 확인
2. 화면이 깨지는 부분이 없는지 확인
3. 주요 기능이 정상 동작하는지 확인
점검 결과를 체크리스트 형식으로 알려주세요.
```

**변경 내용 요약** (`.claude/commands/변경-요약.md`)
```markdown
오늘 변경한 코드를 분석해서 아래 형식으로 요약해 주세요:
- 변경한 파일 목록
- 주요 변경 내용 (3줄 이내)
- 추가로 확인이 필요한 부분
```

> 💡 **Skill과 Commands의 차이**  
> 둘 다 슬래시 명령어로 호출하지만, Skills는 지원 파일(템플릿, 스크립트 등)을 함께 묶어서 관리할 수 있어 더 복잡한 작업에 적합합니다. 간단한 명령어는 Commands, 복잡한 워크플로우는 Skills를 사용하세요.

---

## 12-4. 전체 명령어 빠른 참조

| 명령어 | 하는 일 |
|--------|---------|
| `/help` | 사용 가능한 명령어 목록 보기 |
| `/clear` | 대화 내용 초기화 |
| `/compact` | 대화 내용 압축 (긴 대화 정리) |
| `/recap` | 지금까지 작업 요약 보기 |
| `/config` | 설정 변경 |
| `/mcp` | MCP 서버 관리 |
| `/plugin` | 플러그인 관리 |
| `/memory` | 메모리(CLAUDE.md) 확인 및 수정 |
| `/review` | 코드 리뷰 요청 |
| `/init` | CLAUDE.md 자동 생성 |

> 💡 대화창에서 `/`만 입력하면 자동완성으로 전체 목록을 볼 수 있습니다.

## FAQ

**Q. `/`를 입력했는데 아무것도 안 나와요.**  
Claude Code가 실행 중인지 확인하세요. 터미널에서 `claude`를 입력해서 실행한 뒤 사용해야 합니다.

**Q. `.claude/commands/`와 `.claude/skills/` 중 어디에 만들어야 하나요?**  
간단한 프롬프트 하나면 `commands/`, 여러 파일이나 복잡한 워크플로우가 필요하면 `skills/`를 사용하세요.

**Q. Skill/Commands 명령어와 내장 명령어가 겹치면 어떻게 되나요?**  
내장 명령어가 우선합니다. 파일 이름을 내장 명령어(`clear`, `compact` 등)와 다르게 지어야 합니다.

**Q. 명령어 중간에 취소하고 싶어요.**  
`Control(⌃) + C`를 누르면 실행 중인 명령어를 취소할 수 있습니다.
