# 커스텀 명령어 만들기

> 이 챕터에서 할 것: `.claude/commands/` 폴더에 파일을 저장해서 나만의 슬래시 명령어를 만들어 봅니다.

> ⚠️ 먼저 [6단계: Claude Code 시작하기](06.Claude-Code-시작하기.md)를 완료해야 합니다.

---

## 커스텀 명령어란?

자주 쓰는 작업을 파일로 저장해두면 `/명령어이름`으로 바로 실행할 수 있습니다.  
예를 들어 "배포 전에 이것저것 점검해줘"라는 긴 요청을 매번 타이핑하는 대신, `/배포-점검`만 입력하면 됩니다.

---

## 저장 위치

`.claude/commands/` 폴더에 마크다운(`.md`) 파일을 만들면 됩니다.

```
my-todo-app/
└── .claude/
    ├── commands/    ← 커스텀 명령어 저장 위치
    └── skills/      ← Skill 저장 위치
```

`.claude` 폴더가 없다면 먼저 만들어야 합니다. 자세한 방법은 [Skill 만들기](10.Skill-md-작성하기.md)의 10-2를 참고하세요.

---

## 첫 번째 커스텀 명령어 만들기

`.claude/commands/` 폴더 안에 `테스트.md` 파일을 만들고 아래 내용을 입력하세요:

```markdown
현재 프로젝트의 모든 테스트를 실행하고, 실패한 항목이 있으면 원인과 해결 방법을 알려주세요.
```

`Command(⌘) + S`로 저장합니다.

---

## 사용하기

Claude Code 대화창에서 파일 이름으로 바로 호출합니다:

```
/테스트
```

> 💡 슬래시(`/`) 뒤에 파일 이름(확장자 제외)을 입력하면 됩니다.

---

## 커스텀 명령어 예시

아래를 참고해서 나만의 명령어를 만들어 보세요.

**배포 전 점검** (`.claude/commands/배포-점검.md`)
```markdown
배포 전에 아래 항목을 순서대로 점검해 주세요:
1. 콘솔 오류 메시지가 있는지 확인
2. 화면이 깨지는 부분이 없는지 확인
3. 주요 기능이 정상 동작하는지 확인
점검 결과를 체크리스트 형식으로 알려주세요.
```

**변경 내용 요약** (`.claude/commands/변경-요약.md`)
```markdown
오늘 변경한 코드를 분석해서 아래 형식으로 요약해 주세요:
- 변경한 파일 목록
- 주요 변경 내용 (3줄 이내)
- 추가로 확인이 필요한 부분
```

**코드 정리** (`.claude/commands/코드-정리.md`)
```markdown
현재 열려 있는 파일의 코드를 아래 기준으로 정리해 주세요:
- 사용하지 않는 변수나 함수 제거
- 중복된 코드 합치기
- 변수/함수 이름을 더 명확하게 변경
변경한 내용을 항목별로 설명해 주세요.
```

## Commands vs Skills

둘 다 슬래시 명령어로 호출하지만 용도가 다릅니다.

| | Commands | Skills |
|---|---|---|
| **파일 구조** | `.md` 파일 하나 | 폴더 안에 `SKILL.md` + 지원 파일 |
| **적합한 상황** | 간단한 프롬프트 요청 | 복잡한 워크플로우, 여러 파일 참조 |
| **예시** | `/배포-점검`, `/변경-요약` | `/코드-설명`, `/버그-찾기` |

## FAQ

**Q. 명령어 이름에 한글을 써도 되나요?**  
네, 한글 파일 이름도 됩니다. `/배포-점검` 처럼 그대로 호출할 수 있습니다.

**Q. 내장 명령어(`/clear`, `/compact`)와 이름이 겹치면 어떻게 되나요?**  
내장 명령어가 우선합니다. 겹치지 않는 이름을 사용하세요.

**Q. 수정하면 바로 적용되나요?**  
네, 저장 후 다음 호출 시 바로 반영됩니다.
