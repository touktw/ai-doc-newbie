# 13단계: Agent 만들기

> 이 챕터에서 할 것: 특정 역할에 특화된 나만의 AI Agent를 만들어서 반복 작업을 맡깁니다.

> ⚠️ 먼저 [10단계: Skill.md 작성하기](10.Skill-md-작성하기.md)를 완료하면 더 쉽게 이해할 수 있습니다.

---

## 13-1. Agent란?

Claude Code를 쓰다 보면 같은 종류의 작업을 자주 맡기게 됩니다.  
예를 들어 코드를 수정할 때마다 "보안 문제가 있는지 검토해줘"라고 반복해서 요청하는 경우가 있습니다.

**Agent는 특정 역할에 특화된 별도의 AI**입니다.  
코드 리뷰 전문 Agent, 버그 분석 전문 Agent처럼 역할을 미리 정해두면, 필요할 때 `@코드-리뷰어` 처럼 불러서 사용할 수 있습니다.

---

## 13-2. Skill과 Agent의 차이

비슷해 보이지만 사용 목적이 다릅니다.

| | Skill | Agent |
|---|---|---|
| **역할** | 자주 쓰는 요청을 단축키처럼 저장 | 특정 역할에 특화된 별도의 AI |
| **실행 방식** | 현재 대화에서 실행 | 별도의 대화창에서 독립 실행 |
| **적합한 상황** | 간단한 반복 요청 | 코드 분석처럼 길고 복잡한 작업 |
| **호출 방법** | `/skill-이름` | `@agent-이름` |

> 💡 Skill은 "단축키", Agent는 "전문 직원"으로 생각하면 됩니다.

---

## 13-3. Agent 파일 저장 위치

Agent 파일은 `.claude/agents/` 폴더에 저장합니다.

1. Cursor에서 내 프로젝트 폴더를 엽니다.
2. 이미 만들어둔 `.claude` 폴더 안에 `agents` 폴더를 새로 만듭니다.

폴더 구조:
```
my-todo-app/
└── .claude/
    ├── skills/        ← Skill 파일 저장 위치
    └── agents/        ← Agent 파일 저장 위치 (새로 만들기)
```

---

## 13-4. 첫 번째 Agent 만들기

`.claude/agents/` 폴더 안에 `코드-리뷰어.md` 파일을 만들고 아래 내용을 입력하세요:

```markdown
---
name: 코드-리뷰어
description: 코드 변경 사항을 검토하고 문제점을 찾아줍니다. 코드를 수정한 뒤 리뷰를 요청할 때 사용하세요.
tools: Read, Bash
---

당신은 꼼꼼한 시니어 개발자입니다.
코드를 검토할 때 아래 순서로 진행하세요:

1. 변경된 파일을 확인합니다.
2. 아래 항목을 중점적으로 검토합니다:
   - 오류가 발생할 수 있는 부분
   - 읽기 어려운 코드
   - 보안 문제
3. 발견한 내용을 아래 형식으로 정리합니다:
   - 🔴 반드시 수정: (심각한 문제)
   - 🟡 수정 권장: (개선하면 좋은 부분)
   - 💡 제안: (선택 사항)
```

`Command(⌘) + S`로 저장합니다.

---

## 13-5. Agent 사용하기

Claude Code를 실행한 뒤, `@`를 입력하면 만들어둔 Agent 목록이 나타납니다.

```
@코드-리뷰어 방금 수정한 코드 검토해줘
```

또는 자연스럽게 요청해도 Claude Code가 알아서 적합한 Agent에게 맡깁니다:

```
코드 리뷰 해줘
```

> 💡 `@`를 입력하면 자동완성으로 Agent 목록을 볼 수 있습니다.

## 13-6. 유용한 Agent 예시

아래 내용을 참고해서 나만의 Agent를 만들어 보세요.

**버그 분석가** (`.claude/agents/버그-분석가.md`)
```markdown
---
name: 버그-분석가
description: 오류 메시지나 버그를 분석하고 원인과 해결 방법을 찾아줍니다.
tools: Read, Bash
---

당신은 버그 분석 전문가입니다.
버그를 분석할 때 반드시 아래 순서를 따르세요:

1. 오류 메시지를 정확히 파악합니다.
2. 오류가 발생하는 코드 위치를 찾습니다.
3. 원인을 명확하게 설명합니다.
4. 해결 방법을 단계별로 안내합니다.
```

**문서 작성가** (`.claude/agents/문서-작성가.md`)
```markdown
---
name: 문서-작성가
description: 코드나 기능에 대한 설명 문서를 작성합니다.
tools: Read, Write
---

당신은 기술 문서 전문 작성가입니다.
코드를 분석해서 누구나 이해할 수 있는 설명 문서를 작성합니다.
전문 용어는 쉬운 말로 풀어서 설명하고, 예시를 반드시 포함하세요.
```

## FAQ

**Q. Agent 파일 이름에 한글을 써도 되나요?**  
네, 한글도 됩니다. `@버그-분석가` 처럼 그대로 호출할 수 있습니다.

**Q. Agent가 목록에 안 보여요.**  
`.claude/agents/` 폴더 위치가 맞는지 확인하세요. 파일을 저장한 뒤 Claude Code를 재시작하면 반영됩니다.

**Q. Agent와 Skill을 동시에 써도 되나요?**  
네, 함께 사용할 수 있습니다. 예를 들어 `/기능-추가` Skill로 요청한 뒤 `@코드-리뷰어`로 결과를 검토하는 식으로 활용하면 좋습니다.
