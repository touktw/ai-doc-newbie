# 9단계: CLAUDE.md 작성하기

> 이 챕터에서 할 것: CLAUDE.md 파일을 만들어서 Claude Code에게 내 프로젝트만의 규칙을 알려줍니다.

> ⚠️ 먼저 [6단계: Claude Code 시작하기](06.Claude-Code-시작하기.md)를 완료해야 합니다.

---

## 9-1. CLAUDE.md란?

매번 Claude Code에게 "한국어로 답해줘", "이 앱은 할 일 목록이야"라고 설명하는 것은 번거롭습니다.

**CLAUDE.md는 한 번 써두면 Claude Code가 자동으로 읽는 규칙 파일**입니다.  
프로젝트 폴더에 이 파일을 두면, Claude Code를 실행할 때마다 자동으로 내용을 참고합니다.

---

## 9-2. CLAUDE.md 파일 만들기

1. Cursor에서 내 프로젝트 폴더(`my-todo-app`)를 엽니다.
2. 왼쪽 파일 목록에서 빈 곳을 마우스 오른쪽 버튼으로 클릭합니다.
3. **New File**을 선택합니다.
4. 파일 이름을 `CLAUDE.md`로 입력하고 Enter를 누릅니다.

---

## 9-3. 기본 내용 작성하기

아래 내용을 그대로 복사해서 붙여넣은 후, 내 프로젝트에 맞게 수정하세요:

```markdown
# 프로젝트 규칙

## 프로젝트 소개
할 일 목록 웹앱입니다. HTML, CSS, JavaScript로 만들어졌습니다.

## 응답 언어
항상 한국어로 답변해 주세요.

## 코딩 규칙
- 코드 설명은 한국어 주석으로 작성해 주세요.
- 변수명은 영어로, 주석은 한국어로 작성해 주세요.
- 코드를 수정할 때는 어떤 부분을 왜 바꿨는지 설명해 주세요.
```

`Command(⌘) + S`로 저장합니다.

---

## 9-4. 적용 확인하기

터미널에서 프로젝트 폴더로 이동한 후 Claude Code를 실행합니다:

```bash
cd ~/Desktop/my-todo-app
claude
```

아래처럼 입력해서 CLAUDE.md를 잘 읽었는지 확인해 보세요:

```
이 프로젝트가 어떤 프로젝트인지 설명해줘
```

CLAUDE.md에 적힌 내용을 바탕으로 한국어로 답변이 오면 성공입니다. ✅

---

## 9-5. 유용한 규칙 예시

프로젝트에 맞게 아래 내용을 추가해 보세요:

**응답 스타일 설정**
```markdown
## 응답 스타일
- 전문 용어는 쉬운 말로 풀어서 설명해 주세요.
- 코드 수정 후 어떻게 테스트하면 좋은지도 알려주세요.
```

**금지 사항 설정**
```markdown
## 하지 말아야 할 것
- 기존 기능을 삭제하지 마세요.
- 파일을 새로 만들지 말고 기존 파일을 수정해 주세요.
```

**기술 스택 명시**
```markdown
## 기술 스택
- HTML, CSS, JavaScript만 사용합니다.
- 외부 라이브러리는 추가하지 마세요.
```

## FAQ

**Q. CLAUDE.md 파일 이름이 대소문자가 중요한가요?**  
네, 반드시 `CLAUDE.md`로 대문자로 작성해야 합니다.

**Q. CLAUDE.md를 수정했는데 바로 적용되나요?**  
Claude Code를 종료(`/exit`) 후 다시 실행하면 새 내용이 적용됩니다.

**Q. CLAUDE.md에 무엇이든 쓸 수 있나요?**  
네, 자유롭게 쓸 수 있습니다. 프로젝트 설명, 코딩 스타일, 금지 사항, 자주 쓰는 요청 등 무엇이든 적어두면 도움이 됩니다.
